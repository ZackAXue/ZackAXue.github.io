题目：3. Longest Substring Without Repeating Characters

Given a string s, find the length of the longest substring without repeating characters.

思路

本题使用 sliding Window 的解法，设 left 和 right，用 hashmap 记录字母出现频率。向右移动 right，如果当前字母出现次数不为 0，代表该 no repeating substring 已经到终点，此时右移 left，并减一 map 中的频率，直到 counts[right]频率为 0，则新的 no repeating substring 开始。
时间复杂度为 O(n)，空间复杂度 O(1)。

题目：209. Minimum Size Subarray Sum

Given an array of positive integers nums and a positive integer target, return the minimal length of a contiguous subarray [numsl, numsl+1, …, numsr-1, numsr] of which the sum is greater than or equal to target. If there is no such subarray, return 0 instead.

eg:

思路

Sliding Window

本题最快且最容易理解的写法是使用 sliding window。
建立双指针 left 和 right，移动 right 对 total 进行累加。如果 total 大于等于 target，则触发内部 while 循环，调整 res 之后移动 left, total 减去 left 指向的 value。继续移动 right，直到 right 遍历整个数组。时间复杂度为 O(n)。

Binary Search

使用 Binary Search 写法很有趣。
首先新建数组 total[i]，total 数组为升序。

$total[0] = 0$
$total[i] = nums[0] + nums[1] + … nums[i-1]$

对于每一个 i，利用 Binary Search 找到第一个 j ，满足 j < i 并且 total[j] > total[i] - target。并且易得 total[j - 1] <= total[i] - target

$total[j] > total[i] - target$
则 $nums[0] + nums[1] + nums[j - 1] > nums[0] + nums[1] + nums[i - 1] - target$

$total[j - 1] <= total[i] - target$
则 $nums[0] + nums[1] + … + nums[j - 2] <= nums[0] + nums[1] + … + nums[i - 1] - target$

化简后可得
$target > nums[j] + … +nums[i - 1]$
$target <= nums[j - 1] + … nums[i - 1]$

这意味着对于当前的 i - 1，j - 1 到 i - 1 是最短的满足要求的 subarray。再遍历所有可能的 i 之后，就能找到最短的 subarray。
时间复杂度为 O(nlogn) for * binary search

题目：15. 3Sum

Given an integer array nums, return all the triplets [nums[i], nums[j], nums[k]] such that i != j, i != k, and j != k, and nums[i] + nums[j] + nums[k] == 0.

Notice that the solution set must not contain duplicate triplets.

思路

这道题的思路很简单，先 sort，再设 pivot，之后用双指针找到两数之和等于-pivot。如果遇到合适的 triplet，需要考虑之后的元素是否和当前的相等以避免重复。最后使用一个 while 来避免 pivot 重复。

但是因为 edge case 和一些很蠢的错折磨了很久，写到后面属实有点气急败坏…
几个教训：

  1. a && b 的计算
    先看 a，若 a 是假，则结束，整个表达式的值为假。
    若 a 是真，再看 b，如 b 为假，整个表达式为假。
    若 b 为真，则整个表达式为真。
    所以最后的 while 判断条件如果反过来就会导致[0]这个 case 越界。
  2. 完全没有必要使用指针，两个 int 即可
  3. 关于并列的 if 和 if else 并列区别很大，需要注意。
  4. 不建议使用 for(int i = 1…)这种从 1 开始遍历的写法，很容易想错导致越界访问。并且还是那句话，写之前就想清楚 edge case，最好在纸上列一下。
  5. 最后，气急败坏的状态下建议先做其他事，调整下大脑。

题目：49. Group Anagrams

Given an array of strings strs, group the anagrams together. You can return the answer in any order.

An Anagram is a word or phrase formed by rearranging the letters of a different word or phrase, typically using all the original letters exactly once.

eg: Input: Input: strs = [“eat”,”tea”,”tan”,”ate”,”nat”,”bat”]
Output: [[“bat”],[“nat”,”tan”],[“ate”,”eat”,”tea”]]

思路

使用 count sort，以避免对每一个 str 进行 sort。首先建立一个对于 26 个字母的 count[26]，记录每个字母出现的次数，并且返回一个形式为 a2b2(abab)的 key。使用一个 unorder map 储存 <key,vector>，并且在最后把 vector 导入新建的 vector<vector> res。
时间复杂度为 O(mn) 空间复杂度为 O(26m)，m 为 str 数目，n 为单个 str 长度。
