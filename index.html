题目：15. 3Sum

Given an integer array nums, return all the triplets [nums[i], nums[j], nums[k]] such that i != j, i != k, and j != k, and nums[i] + nums[j] + nums[k] == 0.

Notice that the solution set must not contain duplicate triplets.

思路

这道题的思路很简单，先 sort，再设 pivot，之后用双指针找到两数之和等于-pivot。如果遇到合适的 triplet，需要考虑之后的元素是否和当前的相等以避免重复。最后使用一个 while 来避免 pivot 重复。

但是因为 edge case 和一些很蠢的错折磨了很久，写到后面属实有点气急败坏…
几个教训：

  1. a && b 的计算
    先看 a，若 a 是假，则结束，整个表达式的值为假。
    若 a 是真，再看 b，如 b 为假，整个表达式为假。
    若 b 为真，则整个表达式为真。
    所以最后的 while 判断条件如果反过来就会导致[0]这个 case 越界。
  2. 完全没有必要使用指针，两个 int 即可
  3. 关于并列的 if 和 if else 并列区别很大，需要注意。
  4. 不建议使用 for(int i = 1…)这种从 1 开始遍历的写法，很容易想错导致越界访问。并且还是那句话，写之前就想清楚 edge case，最好在纸上列一下。
  5. 最后，气急败坏的状态下建议先做其他事，调整下大脑。

题目：49. Group Anagrams

Given an array of strings strs, group the anagrams together. You can return the answer in any order.

An Anagram is a word or phrase formed by rearranging the letters of a different word or phrase, typically using all the original letters exactly once.

eg: Input: Input: strs = [“eat”,”tea”,”tan”,”ate”,”nat”,”bat”]
Output: [[“bat”],[“nat”,”tan”],[“ate”,”eat”,”tea”]]

思路

使用 count sort，以避免对每一个 str 进行 sort。首先建立一个对于 26 个字母的 count[26]，记录每个字母出现的次数，并且返回一个形式为 a2b2(abab)的 key。使用一个 unorder map 储存 <key,vector>，并且在最后把 vector 导入新建的 vector<vector> res。
时间复杂度为 O(mn) 空间复杂度为 O(26m)，m 为 str 数目，n 为单个 str 长度。

题目：56. Merge Intervals

Given an array of intervals where intervals[i] = [starti, endi], merge all overlapping intervals, and return an array of the non-overlapping intervals that cover all the intervals in the input.

eg: Input: intervals = [[1,3],[2,6],[8,10],[15,18]]
Output: [[1,6],[8,10],[15,18]]
Explanation: Since intervals [1,3] and [2,6]overlaps, merge them into [1,6].

思路

首先对 interval 进行排序,之后遍历 interval，如果该 interval 是第一个，或者当前的 merged 的尾部和当前 interval 没有相交，直接 push。反之对 merged 尾部的【1】进行修改。注意 A or B 的反为（A 反&B 反 ）
时间复杂度为 O(nlogn) 空间复杂度为 O(n)(sort)
